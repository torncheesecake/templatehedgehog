# Documentation Index

This folder holds product, growth, UX, and technical planning documents.

## Current Documents

- `homepage-blueprint.md`: homepage structure, messaging, and buyer journey notes.
- `homepage-growth-loop.md`: acquisition and conversion loop notes.
- `comparison-section-blueprint.md`: competitive and comparison section planning.
- `technical-proof-blueprint.md`: technical proof and implementation trust strategy.
- `deployment.md`: GitHub Pages and self-hosted Raspberry Pi deployment process.

## Documentation Standards

- Keep durable project operating rules in `AGENTS.md`.
- Keep current health, issues, and technical debt in `CURRENT_STATE.md`.
- Keep executable work queues in `TASKS.md`.
- Keep future architecture or product plans in `PLANS.md`.
- Keep AI-made repository changes in `CHANGELOG_AI.md`.

Planning documents may be more exploratory than code documentation, but they should still state assumptions and avoid unsupported claims.
